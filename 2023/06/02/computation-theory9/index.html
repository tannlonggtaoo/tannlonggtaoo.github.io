介绍一堆经典的NPC问题，以及证明其为NPC的简要思路。

大量NPC问题

Thm. 归约链

  1. SAT ≤p\le_p≤p​ 3SAT ≤p\le_p≤p​ CLIQUE ≤p\le_p≤p​ IS ≤p\le_p≤p​ VC ≤p\le_p≤p​ SC ≤p\le_p≤p​ SubsetSum ≤p\le_p≤p​ 另一种SubsetSum

  2. SC≤p\le_p≤p​HS

  3. 3SAT≤p\le_p≤p​Graph Coloring

注意：这个归约链是可传递的，因此只用证明相邻问题之间可以归约即可。

下面默认归约都指多项式归约，因此省略ppp下标。

归约方法：从AAA多项式时间归约到BBB（A≤pBA\le_p BA≤p​B），就是构造AAA的实例，变换成BBB的实例，且这个变换只用多项式时间就能完成。如果已知AAA为NPC问题且BBB是NP问题（考试时必须写清BBB是NP问题的证明），则根据NPC问题的定义，BBB也是NPC问题。

但必须注意这里实例之间的变换必须是双射，即A,BA,BA,B的实例应该是一一对应的。（考试时必须写清这种一一对应性，可以先正向后反向地证明）

以下问题中的k值是外部给定的，即问题是是否存在一个k阶解，而非找一个最小/大阶的解（后者非平凡的那个方向通常就不是NPC问题了）。

以下给出证明：

SAT ≤p\le_p≤p​ 3SAT ≤p\le_p≤p​ CLIQUE

上一节已证。补充一个max CLIQUE问题（MCP问题，即寻找阶数最大的完全子图），这个就是k没给定而求最大阶解的情形，是NP-hard问题。

CLIQUE≤p\le_p≤p​IS

  • IS：Independent Set problem: 给定一张图G和一个值k，问图中是否存在大小（阶数）为k的独立集。独立集就是顶点之间两两不连边的子图。

    • 归约非常简单：取补图即可。显然是多项式时间能够完成的。

IS≤p\le_p≤p​VC

  • VC：Vertex Cover：给定一张图G和一个值k，问是否存在一个k阶的顶点覆盖

    • k阶顶点覆盖：原图中每条边至少有一个顶点在这k个顶点之中
    • IS归约到VC的方法：记n为给定图的总顶点数，则给定的IS问题“图G中是否存在阶数为k的独立集”可以归约成“图G中是否存在阶数为n-k的顶点覆盖”的VC问题
    • 即如果我们知道VC的n-k阶的解，则取该解相对原图顶点集的k阶补集即为IS，否则n-k阶的解就不是顶点覆盖了（即反证法），反之也类似

VC≤p\le_p≤p​SC

  • SC：Set Cover：给定n阶集合III、III的m个子集S1,S2,...,SmS_1,S_2,...,S_mS1​,S2​,...,Sm​和一个值k，问是否能从m个子集中选出k个使之覆盖原集合III
    • 归约：给定VC的一个实例，则III就定义为图的边集，III的m个子集中的第jjj个子集SjS_jSj​就是VC问题的图中第jjj个顶点所邻接的所有边的集合，这样就可以用解决SC的方法解决VC。这个转换显然是多项式时间能够完成的

SC≤p\le_p≤p​HS

  • HS：Hitting Set：给定n阶集合III、III的m个子集S1,S2,...,SmS_1,S_2,...,S_mS1​,S2​,...,Sm​和一个值k，问能否取定III的一个k阶子集，使得该子集和m个子集中的任意一个的交都非空（所谓“Hitting”）

    • 归约方法：给定SC的一个实例，则将其写成一个二维表，描述了各个子集究竟包含了哪些元素（如下图，1/0表示所在行对应的子集包含/不含所在列对应的元素）：

    • S\I e1e_1e1​ e2e_2e2​ …
      S1S_1S1​ 1 0 …
      S2S_2S2​ 1 1 …
      … … … …

    • 然后，把这张表的01部分转置，就可以作为实例交给HS解决了。显然转置是一一且只需多项式时间的。

SC≤p\le_p≤p​SubsetSum

  • SubsetSum：给定整数w1,...,wnw_1,...,w_nw1​,...,wn​和WWW，问能否可以选出若干整数wiw_iwi​使其和恰好为WWW

    • 注意：这种整数相关问题的输入规模其实是输入各个整数位数之和，即n=∑iceil(log(wi))n=\sum_i ceil(log(w_i))n=∑i​ceil(log(wi​))

    • 另外，问题中要求加起来恰好为WWW，不限定wiw_iwi​的个数（没有指定k），其实和背包问题有紧密联系。

    • 首先证明这个问题是NP的（前面的也要证，不过被省略了）：

      方便起见，引入NP的另一个定义：语言LLL是NP的，当且仅当

      • 存在一个能够检查正确性的多项式时间的DTM MMM
      • 使得x∈Lx\in Lx∈L当且仅当 ∃y,M(x,y)=1\exist y,M(x,y)=1∃y,M(x,y)=1

      MMM可以接受两个参数x,yx,yx,y，其中xxx是正常的待验证是否在LLL中的串，yyy是一个证书，即对解的一个猜测，或者说假设有对应的NTM能解具这个问题，则yyy可以编码成NTM上运行路径的一个编码。

      例如，CLIQUE中一个证书yyy就是任意选择的k个点，此时MMM的任务就是在多项式时间内验证yyy是否是xxx（即x={G,k}x=\{G,k\}x={G,k}）的合法解）

      考试时也可用此方法证明NP性，不过要写明yyy的构造方法和验证算法，且二者必须至多多项式复杂度。

      • 用这个定义，SubsetSum显然是NP的

    • 然后证明NPC：从SC的一个实例（III（设III有n个元素），S1,...,SmS_1,...,S_mS1​,...,Sm​，k）构造一个SubsetSum的实例

      • 首先选一个充分大的进制，如d=nk+m+1d=nk+m+1d=nk+m+1（防止进位）
      • 然后设法构建各个wiw_iwi​（这些wiw_iwi​都是d进制数）

SubsetSum≤p\le_p≤p​另一种SubsetSum

  • SubsetSum的另一种定义：给定整数w1,...,wnw_1,...,w_nw1​,...,wn​，问是否可以将这些整数做一个二分割，使得两组的和相等

    • 归约：给定原始SubsetSum的实例w1,...,wn;Ww_1,...,w_n;Ww1​,...,wn​;W，则构造时分两种情况讨论（设T=∑iwiT=\sum_i w_iT=∑i​wi​）
      • W>T/2W > T/2W>T/2，则可以取w1,...,wn,2W−Tw_1,...,w_n,2W-Tw1​,...,wn​,2W−T，若原始SubsetSum有解，则另一种问题可以按照被选中的“k个数”和“其余数”二分割，且和相等；若另一种SubsetSum问题有解，则假设两个分割各自的和为BBB，于是总和是2B2B2B；但w1+...+wn+2W−T=2Ww_1+...+w_n+2W-T=2Ww1​+...+wn​+2W−T=2W，于是W=BW=BW=B，因此原始SubsetSum有解
      • W≤T/2W\le T/2W≤T/2，则可以取w1,...,wn,T−2Ww_1,...,w_n,T-2Ww1​,...,wn​,T−2W，和上面一样。
